O que são Clients?

(Voltar)

Uma parte essencial do curso foi entender como a rede funciona de verdade: de um lado, o cliente pedindo as informações e, do outro, o servidor respondendo com os dados. Essa dinâmica entre os dois é o que faz tudo acontecer.

O cliente é, basicamente, a pessoa que quer acessar e visualizar algum conteúdo na internet, usando o protocolo TCP/IP para se conectar e navegar.

O que são Servers?

(Voltar)

O servidor é onde as informações ficam guardadas, esperando para serem enviadas ao cliente quando ele quiser ver algo.

Sem essa troca entre cliente e servidor, não teríamos a internet ou redes internas, como a intranet.

Como é uma estrutura básica de um HTML?

(Voltar)

É aqui que a mágica começa: com o HTML!

Embora muita gente pense que o HTML! é uma linguagem de programação, na verdade ele é uma linguagem de marcação. Basicamente, é um arquivo cheio de textos organizados de uma forma que o navegador consegue entender e exibir.

Esses textos organizados são chamados de tags, e cada tag tem uma função e um comportamento específico.

Entendendo a diferença entre listas ordenadas e não ordenadas

(Voltar)

Algo muito útil no HTML são as listas, que ajudam a organizar o conteúdo do site e deixam tudo mais fácil de ler.

No HTML, temos dois tipos de listas: ul e ol.

ul (Unordered List)

Uma lista não ordenada, sem uma sequência numérica – ótima para itens que não precisam de uma ordem específica.

ol (Ordered List)

Uma lista ordenada, com uma contagem sequencial – ideal para passos ou itens que seguem uma ordem.
